## Architecture verification (no violations)

| Rule                                                                                               | Result                                                                      |
| -------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| Signal Engine depends ONLY on `StrategyEvaluator`                                                  | ✓ — zero imports of indicators / `IndicatorRegistry`                        |
| Strategy Evaluators depend on `IndicatorRegistry` (not concrete indicator classes for calculation) | ✓ — resolve by id; ids imported as constants only                           |
| Indicators remain pure mathematical components                                                     | ✓ — no BUY/SELL/HOLD strings in the module                                  |

## Integration observations

Live verification against Binance Spot (2026-07-19):

1. **Provider** — `GET /v1/market/health` → `providerId=binance`, `status=ok`.
2. **Evaluators registered** — inventory `dummy, sma, ema` (dummy remains default).
3. **SMA / EMA / Dummy evaluate** — all returned BUY or SELL via
   `POST /v1/market/signal/evaluate` with correct `metadata.evaluator`.
4. **Determinism** — repeated evaluations produced identical signal,
   confidence, and `indicatorValue` (timestamp excluded).
5. **Cache reuse** — candle cache hits increased on repeated evaluations;
   `providerCalls` did not grow one-for-one with evaluations (cache lifetime).
6. **Paper trading open → close** — positions are **per strategy**. With BTC
   trending such that SMA/EMA/Dummy all emitted SELL, OPEN required a BUY
   elsewhere (Dummy BUY on SOLUSDT@15m). Close used the existing public
   contract: `PATCH` the same strategy’s `parameters.evaluator` to a SELL
   evaluator (SMA), then `POST /paper-trading/execute` → `CLOSE_LONG`.
   Portfolio (`closedPositions=1`) and history (`OPEN_LONG`, `CLOSE_LONG`)
   updated.
7. **No frontend paper-trading UI** — execute/portfolio/history are API-only
   in this baseline; Strategies UI creates/configures strategies only.

## Approved architecture decisions

1. **Bridge module** — only `strategy-evaluators` may know both engines.
2. **No default indicator** — `IndicatorRegistry.resolve` always requires an id.
3. **Dummy remains default evaluator** — first-registered wins (US009).
4. **Confidence** — `min(1, |close − indicator| / close)` (4 dp) until a shared
   `ConfidenceCalculator` is extracted.
5. **Evaluator id vs indicator id** — separate namespaces that currently share
   string values (`sma`/`ema`); configuration uses `parameters.evaluator` for
   the Signal Engine registry and optional `parameters.indicator` for the
   Indicator registry.
6. **Paper positions are per-strategy** — an EMA SELL cannot close an SMA
   position; opposite-signal close must target the same `strategyId`.
